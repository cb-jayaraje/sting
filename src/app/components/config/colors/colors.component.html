Colors are an important element of any design system, as they can be used to convey meaning, create visual interest, and establish brand identity. Here are some guidelines for designing a color palette for a design system:

  • Establish a primary color palette: Choose a set of colors that will be used consistently throughout the design system. These should include a primary brand color and a range of supporting colors that can be used to create visual hierarchy and interest.
  • Use a limited number of colors: It's generally best to use a limited number of colors in a design system, as too many colors can create visual clutter and make it difficult for users to process information.
  • Consider accessibility: Choose colors that are accessible and legible for users with visual impairments. This may include using high-contrast color combinations and avoiding the use of color as the sole means of conveying information.
  • Use color consistently: Use the same colors consistently throughout the design system to create a cohesive visual identity. This includes using the same shades and tones of each color and applying them consistently across different UI elements.
  • Use color to convey meaning: Use color to convey meaning and create visual interest. For example, you might use a bright, bold color to indicate an error or a softer, muted color to indicate a warning.

By following these guidelines, you can create a color palette that is cohesive, accessible, and effective in conveying meaning and establishing brand identity.

Core UI palette

When a Neutral color is used for a UI element, the brightness itself can imply specific significance. When an element turns dark as you hover your mouse, this indicates that the element is clickable. A distinctively bright gray one indicates that the element has been disabled or expired.

Neutral 700 or Neutral 800 is recommended for body text. The soft color of Neutral 700 naturally matches with a background. Therefore it is appropriate for a UI element with concrete function such as menu labels or buttons. If a content itself is empathized as headline, Neutral 800 can make a clearly readable text. Neutral 600 is handy because it is a brighter color compared to the body text yet read as disabled.

We can see the background in white, but if you want to switch modes or separate spaces, light gray such as Neutral 50 or Neutral 100 can be chosen. Until recently, Neutral 50 which is fairly close to white wasn’t used. However, as display technology in digital devices advance and extremely simple designs come in trend, a very bright gray color started to appear in UI. For that reason, Neutral 100 can seem a bit old-fashioned, but it is affordable for low-end devices and users who don’t have clear eyesight. When separating spaces with color over Neutral 50 background, Neutral 100 is useful. You can use darker background color to represent status change such as active, hover for elements like buttons. In this case, 2 levels darker than normal buttons are suggested.

Stroke elements use Neutral 300 and Neutral 400. Neutral 300 is affordable for strokes which divide compartments such as hr. Neutral 400 is appropriate for strokes implying boundary of forms or buttons can be operated.
