Stacked lists are a common design pattern used to display a list of items in a vertical, stacked layout. Here are some guidelines for designing stacked list elements within a design system:

  • Use clear, descriptive labels: The labels for the list items should clearly describe the purpose of each item and the type of content expected
  • Use appropriate visual hierarchy: Make it easy for users to identify the different items in the list and the relationships between them. This can be achieved through the use of visual hierarchy, such as using larger or bolder text for the primary items and smaller, lighter text for secondary items.
  • Consider mobile-friendliness: When designing for small screens, consider how the list items will be displayed. It may be necessary to use a different layout or truncate the labels to ensure that the list is easy to use on a smaller screen.
  • Use appropriate spacing and alignment: Proper spacing and alignment can help to make the list more legible and easier to read. Use consistent spacing between elements and align elements according to the overall layout of the page.
  • Provide clear actions or next steps: If applicable, the list items should clearly communicate any necessary actions or next steps for the user. Make it easy for users to understand what they need to do and how to do it.

By following these guidelines, you can help to create a stacked list experience that is clear, intuitive, and easy to use for your users.
