The design principle of conversation refers to the idea of designing user interfaces that feel like natural, human conversations, rather than mechanical or robotic interactions. This can help users feel more comfortable and engaged with the system, and can make it easier for them to accomplish their goals.

Guidelines

Use natural language

Use language that sounds natural and human, rather than technical or robotic. This can include using contractions, colloquialisms, and other natural language features.

Use appropriate tone

Use an appropriate tone that matches the context and the user's needs and expectations. For example, a chatbot might use a friendly and casual tone in a customer service context, but a more formal tone in a financial or legal context.

Use appropriate pacing

Use a pacing that feels natural and appropriate for the context and the user's needs. This can include using pauses, allowing time for the user to respond, and using appropriate amounts of text or speech.

Use appropriate structure

Use a structure that feels natural and appropriate for the context and the user's needs. This can include using appropriate levels of detail, using appropriate levels of formality, and using appropriate levels of structure (e.g., using lists or bullet points for clear organization).

Test and iterate

Test your conversational user interface with users to ensure that it feels natural and effective, and iterate based on feedback.

Examples

By using conversation and language in design, you can create a more engaging and interactive user experience and improve the overall usability of the application. Few examples below shows how the design principle of conversation can be applied in the Chargebee app.

Chatbots

Chatbots can be used to provide a conversational interface for users to interact with the application and get support or assistance.

Microcopy

Using clear and concise language in microcopy, such as button labels and error messages, can help to create a more conversational and interactive user experience.

Tutorials and onboarding

Using a conversational tone and interactive elements in tutorials and onboarding can help to make the learning process more engaging and interactive for users.

Personalization

Using personalized language and content based on user preferences and behavior can create a more engaging and interactive user experience.
