Guidelines

Input groups are a common design pattern used to group related input fields, such as a username and password, into a single element. Here are some guidelines for designing input group elements within a design system:

  • Use clear, descriptive labels: The labels for the input fields should clearly describe the purpose of each field and the type of input expected.
  • Use appropriate visual hierarchy: Make it easy for users to identify the different input fields within the group and the required format for each field. This can be achieved through the use of visual hierarchy, such as using larger or bolder text for the primary field and smaller, lighter text for secondary fields.
  • Consider mobile-friendliness: When designing for small screens, consider how the input group will be displayed. It may be necessary to use a different layout or truncate the labels to ensure that the input group is easy to use on a smaller screen.
  • Use appropriate spacing and alignment: Proper spacing and alignment can help to make the input group more legible and easier to use. Use consistent spacing between elements and align elements according to the overall layout of the page.
  • Provide clear feedback: When the input group is activated, provide clear feedback to the user to indicate the status of the input. This can be achieved through the use of visual or auditory cues.

By following these guidelines, you can help to create an input group experience that is clear, intuitive, and easy to use for your users.
