The design principle of feedback refers to the idea that an application or system should provide users with information about the actions they have taken and the results of those actions. This can help users understand what is happening, identify any issues that may arise, and feel more in control of the system.

Guidelines

Make feedback timely

Feedback should be provided as soon as possible after an action has been taken, so users can quickly understand the consequences of their actions.

Make feedback relevant

Feedback should be relevant to the action that was taken, so users can understand how their actions are affecting the system.

Make feedback clear

Feedback should be presented in a clear and concise way, using language and formatting that is easy to understand.

Make feedback actionable

Feedback should provide users with information that they can use to take further action, such as by providing them with options for how to resolve an issue or move forward.

Make feedback consistent

Feedback should be consistent across the application or system, so users can easily understand how to interpret and use the information they receive.

Examples

By providing clear and timely feedback to users, you can help to improve the overall user experience and reduce frustration. Few examples below shows how the design principle of feedback can be applied in the Chargebee app.

In-line validation

When a user is entering data into a form, providing immediate feedback on whether the data is valid can help to reduce frustration and improve the overall user experience.

Loading indicators

When a user initiates an action that requires the application to load data or perform some other task, a loading indicator can provide feedback to the user that the action is being processed.

Cannot delete the only payment method for this account. Setup a backup payment method to delete the current one.

Error messages

If an error occurs while a user is interacting with the application, providing a clear and concise error message can help the user understand what went wrong and how to fix it.

The customer has been created successfully. You can now add subscriptions to the customer.

Confirmation messages

When a user performs an action that has significant consequences, such as deleting a file or making a purchase, a confirmation message can provide feedback to the user that the action has been completed.
