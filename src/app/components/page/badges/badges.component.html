Guidelines

Badges are a common element in user interfaces and are used to display additional information or highlight specific content. When designing badges for a design system, it's important to consider the following guidelines:

  • Use badges sparingly: Badges should be used sparingly, as they can be visually distracting and draw attention away from other content. It's generally best to limit the use of badges to only the most important or relevant information.
  • Use clear and concise labels: The labels for badges should be clear and concise, using as few words as possible. Avoid using ambiguous or vague labels.
  • Use a consistent badge style: The style of the badges should be consistent throughout the design system. This includes the font, color, and overall layout of the badges.
  • Use active and inactive states: It should be clear to users which badges are currently active and which are inactive. This can be achieved through the use of visual indicators such as bold or underlined text or a different background color.
  • Consider the context in which badges are used: The design of badges should be appropriate for the context in which they are used. For example, badges used to highlight errors or warnings should have a different visual style than badges used to highlight new or popular content.

By following these guidelines, you can create an effective and intuitive badge interface that helps users quickly understand and identify important information.
